Курсовой проект по курсу SQL.

	База данных предназначена для хранения результатов футбольных матчей в европейских турниров. Она предоставляет базовые статистики турниров и команд, а так же осуществляет "навный" прогноз на предстоящие игры, исходя из исторических данных.
	Используется загрузка данных из файлов, необходимо включить local_infile, а так же переписать пути которые будут у вас. Понимаю, что переписывать пути не очень удобно, по этому так же прикрепляю дамп базы.

Структура:
	1. Талицы:
		а) catalog - каталог наименований справочников.
		б) catalog_data - наполнение справочников, заложена многоуровневая структура.
		в) club - таблица наименований футбольных клубов с привязкой к стране.
		г) tournament - таблица турниров с привязкой к стране и типу турнира.
		д) calendar - расписание игр сезона, привязано к турнирам, клубам и статусу игры.
		е) result - результаты сыгранных матчей, привязаны к календарю.
		ж) club_statistic_before_match - базовые статистики играющих в конкретном матче комад собранные из исторических данных и прогноз на текущую игру, привязано к календарю.
	2. ПредставленияЖ
		а) tourn_table - турнирная таблица по результатам сыгранных матчей.
		б) tournament_statistics - базовые статистики турниров по результатам сыгранных матчей.
	3. Процедуры:
		а) calc_club_statistics - процедура расчета статистик играющих команд и обновляющая заносящая эти данные в club_statistics_before_match.
	4. Триггеры:
		а) sequence_catalog_data - определяет порядковые номера подкаталогов в catalog_data.
		б) insert_club_statistics - наполняет club_statistics_before_match первичными ключами из календаря.
		в) club_statistics - вносит рассчитанные с помощью процедуры calc_club_statistic данные в club_statistics_before_match. Данный триггер не нужен, т.к. внутреннюю функцию лучше вызывать руками из-за её тяжеловесности, но здесьб присутствует для более удобного автоматического наполнения БД.

Файлы:
	1. foot_stat.sql - скрипт создания и наполнения базы(Важно учесть описанную вначале загрузку из файлов).
	2. foot_stat_dump.sql - дамп базы данных.
	3. readme.txt - это файл с описанием.
	4. Tournaments.csv - данные о чемпионатах.
	5. Clubs.csv - данные о клубах.
	6. Calendar.csv - данные о расписании игр.
	7. Results.csv - данные о результатах сыгранных матчей.
	8. foot_stat_ER - ER диаграмма.
